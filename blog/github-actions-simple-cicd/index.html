Github Actions - simple CI/CD

Dec 29, 2021 | 3 minutes read
Share this: Twitter Facebook

Tags: general

Hello fellows!

Good to see you all. It was a really busy time but it happened - finally found time to write some post about my observations related with GitHub Actions as a good CICD solution that costs almost nothing.

Why do I need CI/CD?

That’s a good question though. First you need to think if:

  • you have some tasks like some unit / integration tests you would like to run every time you push new commit to the branch
  • it takes you a lot of time to run all the commands and do the app release
  • you want to verify each time your commits quality is good enough (e.g. properly formatted files, no secrets in repository files and so on)

Of course there is a lot more reasons you need CI/CD so that’s a good approach to make yourself familiar with the terminology, some use cases, pros and cons of that.

If you do need a CI/CD then…

…you are starting to google things pretty rapidly, searching through StackOverflow.com or reading some articles about TOP 10 CI/CD solutions. That’s a good idea but please - don’t lose yourself in that and if you don’t have an idea - maybe start from something easy to learn like GitHub Actions? If you decide that doesn’t have some features you need then you can move to other solutions.

GitHub Actions - first impressions

To be honest - I’m not that advanced in GH Actions as you might guess. But thanks to really good docs and samples the learning curve is not so sharp. If you used any other CICD tool like Jenkins or Travis CI - you will love the UI and configuration steps in GitHub Actions.

Users audience and pricing

It’s hard to say what is the users audience of that tool. It will be a great solution for small projects or developers maintaining own side on GitHub Pages like me. I’m really keen to see that solution in some bigger project (example with many Docker images to be build, some unit / integration tests, deployments to real test and prod environments).

When we move to the pricing - it’s great that we’re getting getting some compute resources for free (amount depends on product you are using). We need to be more cereful when we’re targeting really high workload and our CICD will have a lot of tasks to handle. In that situation - we can use some calcula provided by GitHub to find cost estimate.

Summing up - worth it

GitHub Actions is a really good solution for people who are not familiar with CI/CD concept and want to learn it. It’s also a good choice for a small projects. I would highly recommend to find some success stories or posts when you are thinking about moving to the GitHub Actions with mid / big size projects.
